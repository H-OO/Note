## Base64的由来

**常用ASCII表**

* 48~57 代表 0-9
* 65~90 代表 A-Z
* 97~122 代表 a-z

**完整Base64表**

*  0~25 代表 A-Z
* 26~51 代表 a-z
* 52~61 代表 0-9
* 62 代表 +
* 63 代表 /

```
Girl

【ASCII】
=========================================
   文本    ASCII编码   ASCII编码转二进制   
=========================================
|   G   |     71    |     0100 0111     |
|   i   |    105    |     0110 1001     |
|   r   |    114    |     0111 0010     |
|   l   |    108    |     0110 1100     |

【Base64】
1、将Girl文本的二进制编码进行排列
============================================↓↓
0100 0111 0110 1001 0111 0010 0110 1100
============================================↓↓
2、每6个bit为一组（若最后一组不足6个bit则用0补充）
010001 110110 100101 110010 011011 00[0000]
============================================↓↓
3、将每组转成十进制编码
|   010001   |    17    |
|   110110   |    54    |
|   100101   |    37    |
|   110010   |    50    |
|   011011   |    27    |
|   000000   |     0    |
============================================↓↓
4、查询十进制编码对应base64编码表的文本
整体为8的倍数
|    17    |     R     |
|    54    |     2     |
|    37    |     l     |
|    50    |     y     |
|    27    |     b     |
|     0    |     A     |
Base64的最小单位为8个字符，不足8个字符则用=补充
|          |     =     |
|          |     =     |
============================================↓↓
Gril转Base64的结果为：R2lybA==
```
